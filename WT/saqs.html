1. Design a PHP program to print reverse of any number

<?php
function reverseNumber($num) {
    // Convert number to string
    $numStr = strval($num);

    // Reverse the string
    $reversedStr = strrev($numStr);

    // Convert back to number
    $reversedNum = intval($reversedStr);

    return $reversedNum;
}

// Sample input
$input = 12345;
echo "Original number: $input\\n";
echo "Reversed number: " . reverseNumber($input) . "\\n";
?>

Running the above code with an input of 12345 will output: 

Original number: 12345
Reversed number: 54321

2. Define PHP Function.

PHP Function Definition

  • Purpose: A PHP function is a block of code designed to perform a specific task, which can be reused throughout the program.

Syntax:
function functionName(parameters) { // code to be executed }

  • Example:

    <?php
function addNumbers($a, $b) {
    return $a + $b;
}

?>

Key Points:

  • Declaration: Use the function keyword followed by the function name and parentheses.
  • Parameters: Optional. Functions can take parameters (arguments) to process.
  • Return Value: Use return to send a result back to the calling code.
  • Reusability: Functions allow code to be reused and organized logically.

5. Design a PHP code to swap any two numbers

<?php
function swapNumbers(&$a, &$b) {
    $temp = $a;
    $a = $b;
    $b = $temp;
}

// Sample input
$number1 = 5;
$number2 = 10;

echo "Before swapping: \\n";
echo "Number 1: $number1\\n";
echo "Number 2: $number2\\n";

swapNumbers($number1, $number2);

echo "After swapping: \\n";
echo "Number 1: $number1\\n";
echo "Number 2: $number2\\n";
?>

Output

When running the above code with number1 = 5 and number2 = 10, the output will be: 

Before swapping:
Number 1: 5
Number 2: 10
After swapping:
Number 1: 10
Number 2: 5

7. Write the structure of PHP script with an example.

Structure of a PHP Script

A PHP script typically follows a basic structure that includes the following components:

  1. PHP Opening Tag: All PHP code is enclosed within <?php ... ?> tags.
  2. Variables and Constants: Declaration of variables and constants.
  3. Functions: Definition of functions.
  4. Control Structures: Use of control structures like if-else, loops, etc.
  5. Output Statements: Generating output using echo or print statements.

Example:

<?php
// PHP Opening Tag

// Variables
$greeting = "Hello, World!";
$number1 = 5;
$number2 = 10;

// Constants
define("PI", 3.14);

// Function Definition
function addNumbers($a, $b) {
    return $a + $b;
}

// Control Structures
if ($number1 < $number2) {
    echo "$number1 is less than $number2\\n";
} else {
    echo "$number1 is not less than $number2\\n";
}

// Function Call
$sum = addNumbers($number1, $number2);

// Output Statement
echo "Sum of $number1 and $number2 is: $sum\\n";
echo "Greeting: $greeting\\n";
echo "Value of PI: " . PI . "\\n";
?>

8. List various String Functions in PHP.

Various String Functions in PHP:

Length and Character Functions:

  • strlen($string): Returns the length of a string.
  • mb_strlen($string): Returns the length of a string using multibyte characters.

Case Manipulation:

  • strtolower($string): Converts a string to lowercase.
  • strtoupper($string): Converts a string to uppercase.
  • ucfirst($string): Converts the first character of a string to uppercase.
  • ucwords($string): Converts the first character of each word in a string to uppercase.

Substring Functions:

  • substr($string, $start, $length): Returns a part of a string.
  • mb_substr($string, $start, $length): Returns a part of a string using multibyte characters.
  • strpos($haystack, $needle): Finds the position of the first occurrence of a substring in a string.
  • str_replace($search, $replace, $subject): Replaces all occurrences of a search string with a replacement string.

Trimming and Padding:

  • trim($string): Removes whitespace or other characters from the beginning and end of a string.
  • ltrim($string): Removes whitespace or other characters from the beginning of a string.
  • rtrim($string): Removes whitespace or other characters from the end of a string.
  • str_pad($string, $length, $pad_string, $pad_type): Pads a string to a certain length with another string.

Encoding and Decoding:

  • urlencode($string): URL-encodes a string.
  • urldecode($string): Decodes URL-encoded string.
  • htmlentities($string): Converts characters to HTML entities.
  • htmlspecialchars($string): Converts special characters to HTML entities.

Formatting and Checking:

  • sprintf($format, $args...): Returns a formatted string.
  • printf($format, $args...): Outputs a formatted string.
  • strcmp($str1, $str2): Compares two strings.
  • strrev($string): Reverses a string.

Regular Expressions:

  • preg_match($pattern, $subject, $matches): Performs a regular expression match.
  • preg_replace($pattern, $replacement, $subject): Performs a regular expression search and replace.

Usage Example:

<?php
$string = "Hello, World!";

echo "Length of string: " . strlen($string) . "\\n";
echo "Uppercase: " . strtoupper($string) . "\\n";
echo "Lowercase: " . strtolower($string) . "\\n";
echo "Substring: " . substr($string, 0, 5) . "\\n";
echo "Replace: " . str_replace("World", "PHP", $string) . "\\n";
?>

11. Define XML? What are the advantages of XML.

XML Definition

XML (Extensible Markup Language) is a markup language designed to store and transport data. It provides a flexible format for creating structured documents and data interchange on the internet.

Advantages of XML

  1. Platform-Independent:
    • XML documents can be processed on any platform using various programming languages and tools.
  2. Human-Readable and Self-Descriptive:
    • XML uses tags to define data elements and attributes, making it easy for humans to read and understand.
  3. Supports Hierarchical Structure:
    • XML allows nesting of elements within elements, creating a hierarchical structure that can represent complex relationships.
  4. Interoperability:
    • XML enables data interchange between different systems and applications, facilitating interoperability.
  5. Customizable:
    • XML documents can define custom tags and attributes, allowing developers to create specific data structures tailored to their needs.
  6. Compatibility with Existing Standards:
    • XML integrates well with other standards like XSLT (for transforming XML data) and XML Schema (for defining the structure and constraints of XML documents).
  7. Extensibility:
    • XML documents can be extended with additional elements and attributes without breaking compatibility with existing applications.

12. Explain about the purpose of DTD.

Purpose of DTD (Document Type Definition)

  • Definition: DTD defines the structure and content rules for XML documents.
  • Key Purposes:
    1. Document Structure: Specifies elements, attributes, and their relationships.
    2. Data Integrity: Ensures consistency and adherence to standards.
    3. Validation: Validates XML documents against predefined rules.
    4. Interoperability: Promotes compatibility between systems.
    5. Reuse: Facilitates modular document design and component reuse.
    6. Documentation: Provides clear guidelines and reference for XML structure.

Example of a Simple DTD

<!DOCTYPE library [
    <!ELEMENT library (book+)>
    <!ELEMENT book (title, author, genre, published_year)>
    <!ELEMENT title (#PCDATA)>
    <!ELEMENT author (#PCDATA)>
    <!ELEMENT genre (#PCDATA)>
    <!ELEMENT published_year (#PCDATA)>
]>
  • <!DOCTYPE ...>: Specifies document type.
  • <!ELEMENT ...>: Defines elements and their content.
  • (book+): Indicates one or more <book> elements in <library>.
  • (#PCDATA): Denotes parsed character data (text content).

28. What is the scope of the variables in JavaScript.

Scope of Variables in JavaScript

  • Global Scope:
    • Variables declared outside of any function.
    • Accessible throughout the script, including inside functions.
  • Local Scope:
    • Variables declared inside a function using var, let, or const.
    • Accessible only within that function and not outside.
  • Block Scope (ES6):
    • Variables declared with let and const have block-level scope.
    • Limited to the block (enclosed by {}) where they are defined.
  • Function Parameters:
    • Parameters passed into functions act as local variables within the function.

JavaScript variables have function-level scope by default, but ES6 introduced block-level scope for variables declared with let and const. Understanding scope helps manage variable access and prevents unintended modifications or conflicts.

29. Define Event. How events are handled in JavaScript.

Event Definition and Handling in JavaScript

Event Definition:

  • Event: In JavaScript, an event is an action or occurrence (like a mouse click, key press, or page load) that can be detected and responded to by the browser or web application.

Handling Events:

  • Events are handled in JavaScript by attaching event listeners to HTML elements or the document itself.

  • Example:

    // Adding an event listener to a button element
document.getElementById("myButton").addEventListener("click", function() {
    // Event handling code here
    console.log("Button clicked!");
});

  • The addEventListener method is used to specify the event type (click in this case) and a function to execute when the event occurs.

Types of Events:

  • Common events include mouse events (click, mouseover, mouseout), keyboard events (keydown, keyup), form events (submit, change), and document loading events (DOMContentLoaded, load).

Event Propagation:

  • Events in JavaScript follow a propagation model where events can propagate through the DOM hierarchy from the target element to its ancestors (event bubbling) or from ancestors to the target element (event capturing).
  • Event propagation can be controlled using methods like stopPropagation().

32. Explain about Servlet API.

Servlet API Overview

  • Definition: Set of Java interfaces and classes for creating servlets, which are Java programs that run on a web server.
  • Functionality:
    • Handles client requests (HTTP) and generates dynamic responses.
    • Manages servlet lifecycle (init(), service(), destroy()).
    • Provides HttpServletRequest and HttpServletResponse for request and response handling.
    • Supports session management (HttpSession) and cookie handling (Cookie).
  • Key Components:
    • Servlet Interface: Defines servlet methods.
    • GenericServlet Class: Abstract class for servlet implementation.
    • HttpServlet Class: Extends GenericServlet with HTTP-specific functionality.
  • Usage:
    • Deployed in web applications (.war files).
    • Managed by servlet containers (like Tomcat) in Java EE or Jakarta EE environments.

The Servlet API enables Java developers to create dynamic web applications by handling HTTP requests, managing sessions, and generating dynamic content on web servers.

33. How to create a cookie using servlet

  • Create a Cookie object: Cookie cookie = new Cookie("cookieName", "cookieValue");
  • Set attributes: You can set attributes like:
    • cookie.setMaxAge(seconds); to define the cookies lifetime.
    • cookie.setPath("/"); to specify the path for which the cookie is valid.
    • cookie.setDomain(".example.com"); to specify the domain for which the cookie is valid.
  • Add the cookie to the response: response.addCookie(cookie);
  • Retrieve the cookie from the request: Cookie[] cookies = request.getCookies();
  • Access cookie data: Iterate through the cookies array and use cookie.getName() and cookie.getValue() to access data.

36. Discuss about Http Request

  • Client-to-server communication: An HTTP request is sent from a client (e.g., web browser) to a web server.
  • Contains request headers and body: The request includes headers containing information like the requested URL, HTTP method, and cookies, and a body that can hold data submitted by the client.
  • Different HTTP methods: Common methods include GET (retrieve data), POST (submit data), PUT (update data), DELETE (remove data), etc.
  • Used by servlets to receive information: Servlets use the HttpServletRequest object to access information from the incoming request, such as URL parameters, headers, and the request body.

37. What is Session?

  • Maintains user state across multiple requests: A session represents a users interaction with a web application over a period of time.
  • Stores user-specific information: It allows the server to store data related to a specific user, like shopping cart items or login details.
  • Managed by the server: Sessions are typically managed by the web server and identified using a unique session ID.
  • Ends after inactivity: A session usually expires after a certain period of inactivity or when the user explicitly logs out.
  • Used for personalized experiences: Sessions enable applications to provide personalized experiences by tracking user preferences and actions.

38. Discuss about Http Responses

  • Server-to-client communication: An HTTP response is sent from the server back to the client (e.g., web browser).
  • Includes response headers and body: The response consists of headers containing information like status codes (e.g., 200 OK, 404 Not Found), cookies, and a body containing the requested data.
  • Status codes indicate response status: Status codes provide information about the success or failure of the request.
  • Content type determines response format: The Content-Type header specifies the format of the response body, such as HTML, JSON, or images.
  • Generated by servlets: Servlets use the HttpServletResponse object to construct and send the response back to the client.

39. What are the different types of session tracking mechanism supported by Servlets?

  • URL Rewriting: The session ID is appended to the URL of each requested resource.
  • Cookies: A unique session ID is stored as a cookie on the clients browser.
  • Hidden Form Fields: The session ID is embedded as a hidden form field in HTML forms.
  • HTTPSession: Provides a built-in API to manage sessions using cookies and server-side data storage.

40. Explain about Common gateway interface (CGI).

  • Early technology for dynamic web content: CGI allows web servers to execute external programs (typically scripts) to generate dynamic content.
  • Script is invoked for each request: Every time a CGI program is called, a new process is created, leading to potential performance issues with heavy traffic.
  • Limited to specific languages: CGI typically relies on scripting languages like Perl or Python.
  • Replaced by newer technologies: CGI has been largely replaced by more efficient technologies like servlets, PHP, and ASP.NET.
  • Still used in some legacy applications: Some older web applications may still rely on CGI for specific functionalities.

41. Define JSP expression.

  • JSP expressions are used to embed Java code directly into JSP pages.
  • They are enclosed within <%= %> tags.
  • The code inside the expression is evaluated at runtime and the result is inserted into the HTML output.
  • Example: <%= "Hello, " + name %> will display "Hello, " followed by the value of the name variable.

42. What are the directives in JSP with syntax?

  • Page Directive: <%@ page ... %>
    • Specifies page-level attributes like language, import, error page, etc.
    • Example: <%@ page language="java" import="java.util.*" %>
  • Include Directive: <%@ include file="filename.jsp" %>
    • Includes another JSP page at the time of translation.
  • Taglib Directive: <%@ taglib prefix="c" uri="<http://java.sun.com/jsp/jstl/core>" %>
    • Declares custom tag libraries for use within the JSP page.

43. What is the syntax of JSP?

  • JSP syntax combines HTML with embedded Java code.
  • HTML elements define the page structure and content.
  • Java code is embedded within JSP tags (<% ... %>, <%= ... %>, <%! ... %>) to provide dynamic content.
  • The JSP compiler translates the page into a servlet, which generates the final HTML output.

44. Explain about Scriptlet Tag.

  • Scriptlet tags are used to execute Java code within JSP pages.
  • They are enclosed within <% ... %> tags.
  • The code inside the scriptlet is executed every time the JSP page is requested.
  • Example: <% for(int i=0; i<10; i++) { out.println(i + " "); } %> will print numbers from 0 to 9.

45. What are implicit objects in JSP?

  • Implicit objects are pre-defined objects available within JSP pages without explicit declaration.
  • They provide access to the servlet API and the request/response context.
  • Some common implicit objects include:
    • request: Holds request data from the client.
    • response: Used to send response back to the client.
    • out: Used for writing output to the response stream.
    • session: Used to manage user sessions.

46. Explain about the methods of getProperty() and setProperty().

  • getProperty() and setProperty() are methods associated with the java.util.Properties class.
  • They are used to read and write properties from a property file.
  • getProperty(String key): Returns the value associated with the specified key.
  • setProperty(String key, String value): Sets the specified key-value pair in the properties object.

47. What is JSP?

  • JSP (JavaServer Pages) is a technology for creating dynamic web pages.
  • It allows developers to embed Java code within HTML pages to generate dynamic content.
  • JSP pages are compiled into servlets, which handle requests and generate responses.

48. What are the Beans in JSP Page.

  • JSP beans are Java classes that represent business logic or data.
  • They are typically used to encapsulate reusable components and improve code organization.
  • They can be accessed from JSP pages using the jsp:useBean tag.
  • Example: A Customer bean can encapsulate customer information like name, address, and phone number.

49. What are the difference between JSP and HTML.

Feature JSP (JavaServer Pages) HTML (HyperText Markup Language)
Dynamic Content Uses Java code embedded in HTML-like tags (<% %>). Static markup language for creating web pages.
Execution Runs on the server-side, processed into servlets. Interpreted by the browser on the client-side.
File Extension .jsp .html, .htm
Java Integration Allows embedding Java code for server-side processing. Limited to static content without Java integration.
Server-side Processing Processed by servlet containers like Tomcat. Rendered directly by browsers without server processing.

50. How are Cookies used for Session tracking in JSP.

  • Cookies are small text files stored on the users computer by the web server.
  • JSP can use cookies to store session information (e.g., user ID, shopping cart contents) across multiple requests.
  • The server sends a cookie with a unique identifier to the users browser on the first request.
  • The browser sends the cookie back with every subsequent request.
  • The server uses the cookie identifier to identify the users session and retrieve the associated data.
  • Example: A shopping cart application might use cookies to store the items added to the cart between different page views.
